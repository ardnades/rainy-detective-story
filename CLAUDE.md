# 寫作憲法 — 雨夜偵探社：第七封情書

本檔是本專案生成任何劇情文字（信件、對話、SNS 文案、結局演出）時的最高規則。
與全域偏好衝突時，以本檔為準。

## 三大原則

1. **先查台帳，再動筆**：寫任何劇情前必讀 `brief/canon-log.md`（確定事實）與 `brief/info-matrix.md`（誰在哪天知道什麼）。角色不可說出他此刻不該知道的資訊。
2. **寫完必過檢查**：任何新增/修改的劇情文字，必須跑 `scripts/check-text.ps1`，違規清零前視為未完成。
3. **事實只增不改**：寫作中新確定的事實（日期、地名、數字、人物關係）必須追記到 `canon-log.md`。已凍結的事實不可改寫；真要改，須在台帳留改訂紀錄並說明理由。

## 正典版本

- **現行正典 = v2（白坂遙版）**：凶手桐嶋宗一郎、共犯儀武湊、死者白坂遙。詳見 `brief/canon-log.md`。
- v1（白瀨千夏版，`brief/season1.md`）已廢棄，僅供結構參考。不可混用 v1 的人名、線索、結局條件。

## 文字規範（繁體中文）

- 省略號統一用「……」（六點）。禁用「...」「。。。」「…」單三點。
- 破折號統一用「——」（雙字）。禁用「--」「—」單字。
- 劇情本文使用全形標點（，。？！：；「」）。半形標點只允許出現在數字、代號、英文內（如 B-17、0311）。
- 禁止簡體字混入。
- JSON 內的劇情文字（story.json、endings.json 等）不得含 Markdown 記號（`**`、`##`、反引號）。
- 引號層級：第一層「」，第二層『』。

## AI 慣用語黑名單

劇情本文禁用以下詞（機械檢查會抓）：

> 不禁、彷彿、宛如、深邃、璀璨、一抹、眸子、嘴角勾起、嘴角上揚、緩緩道、緩緩說、心中一震、心頭一緊、湧上心頭、若有所思、莫名地、難以言喻、千言萬語、時光荏苒、歲月靜好

需要同義表達時，改用具體動作或感官描寫代替。

## 角色語氣

- 每個有台詞的角色須有語氣表（雛形：`templates/character-sheet.md`），定義一人稱、句尾習慣、禁則語、知覺癖。
- 生成該角色台詞前先讀其語氣表；新角色開口前先建表。
- 白坂遙只透過信件存在，信件文體比照已完成的七封信（收錄於 canon-log 附錄索引），不可出現她本人視角的場景。

## 工作流程（每次寫作循環）

1. 查 `canon-log.md` + `info-matrix.md`，確認本段涉及的事實與資訊邊界
2. 讀相關角色語氣表
3. 生成文字
4. 跑 `pwsh scripts/check-text.ps1 <檔案>`，違規清零
5. 新確定的事實追記 `canon-log.md`；新揭露的資訊更新 `info-matrix.md`

## 安全底線（沿用全案規則）

- 主線真相不鎖付費牆；付費只補完演出
- 所有角色 18 歲以上；含虛構標示
- 前端零 API key、零即時 AI 呼叫
